🕹️ Xonix Game – Data Structures Project

A fully-featured C++ arcade-style game developed from scratch as part of the Data Structures Course at FAST University Islamabad. Showcasing custom implementations of core data structures integrated into engaging gameplay with SFML.

🎯 Project Goals

  • Apply core Data Structures concepts in a real-world game setting
  • Build a custom backend for user handling, scoring, matchmaking, and gameplay
  • Showcase modular, clean, and well-documented C++ code
  • Create a multi-mode game with persistent data and social features

🧩 Features

  • 🎮 Classic Xonix Gameplay – Draw lines to capture territory and avoid enemies
  • 🧑‍🤝‍🧑 Single & Multiplayer Modes – Solo play or local two-player co-op
  • 🔐 Login & Registration System – Secure, file-based authentication
  • 📊 Scoring & Bonuses – Multipliers, rewards, and power-ups
  • 👤 Player Profiles – Username, points, history, inventory, themes
  • 🤝 Friend System – Requests handled with linked lists
  • 🏆 Leaderboard – Top 10 players tracked with a Min-Heap
  • 🤖 Matchmaking – Priority Queue for score-based pairing
  • 🚪 Game Room Simulation – Queue-based player pairing
  • 💾 Save & Load Game – Persistent game states with resume capability
  • 🎨 Inventory & Themes – Custom themes managed with an AVL Tree

🧱 Data Structures Implemented

Feature Data Structure
Game Board2D Array
Player & EnemiesDynamic Array
Friend SystemLinked List
Fast Player LookupHash Table
LeaderboardMin-Heap
MatchmakingPriority Queue
Game RoomQueue
Themes / InventoryAVL Tree
Game Save / LoadLinked List + File I/O

🛠️ Built With

  • 💻 Language: C++
  • 🎨 Graphics: SFML (Simple and Fast Multimedia Library)
  • 📁 Persistence: File handling (structured text files)
  • 🧠 Custom Data Structures: Implemented from scratch (no STL)

🚀 How to Run

  1. 📥 Download or clone the repository
  2. 📂 Extract the folder and open the .sln file in Visual Studio
  3. 🔧 Set configuration to Release Mode
  4. ▶️ Build and Run the project
  5. 🧩 Ensure SFML is properly linked (already configured in the solution)
